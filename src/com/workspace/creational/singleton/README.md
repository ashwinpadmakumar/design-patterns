# Singleton Design Pattern
```
Singleton pattern is a design pattern which is used to create only one instance of an object.
It is used in the cases of creating a logger or session or creating a database instance
```

### Conditions
```
- Constructor should be private.
- A public and static method to return the instance.
- Instance variable should be private and static.
```

### Types
```
- Eager Initialization
- Lazy Initialization
- Thread safe Method Level Synchronization
- Thread save Block Level Synchronization
```
